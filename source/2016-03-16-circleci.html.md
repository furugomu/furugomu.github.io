---
title: CircleCI が middleman blog を github pages にデプロイしてくれるための設定
date: 2016-03-16
---

## circleci.yml

```yaml
# master だけ対象にする (source/circle.yml にも書く)
general:
  branches:
    only:
      - master

# mime-types-data とやらが ruby 2 を要求するので 2.3.0 にしてやる
machine:
  ruby:
    version: 2.3.0

# テストすることは無いので true コマンドを実行する
test:
  override:
    - 'true'

deployment:
  production:
    branch: master
    commands:
      # user.name と user.email は git commit の時に必要
      - git config --global user.name "circleci"
      - git config --global user.email "furugomu@gmail.com"
      - bundle exec middleman deploy -e production
```

## source/circle.yml

```yaml
general:
  branches:
    ignore:
      - gh-pages
```

## Gemfile

middleman-deploy の正式版は middleman 4 に対応していないためアルファ版を使う。

```ruby
gem 'middleman-deploy', '~> 2.0.0.pre.alpha'
```

## CircleCI に秘密鍵を登録

CircleCI は GitHub リポジトリの読み取り権限しか持っていないので middleman deploy に失敗する。
なので秘密鍵を登録して書けるようにしてやる。

```sh
ssh-keygen -f ~/.ssh/circleci-github
```

https://github.com/settings/ssh で ~/.ssh/circleci-github.pub を登録する。

https://circleci.com/gh/furugomu/furugomu.github.io/edit#ssh で ~/.ssh/circleci-github を登録する。
